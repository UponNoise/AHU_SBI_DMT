## Chapter 1: Introduction

### Key Knowledge Points
- **Basic Concepts:** Computer network as a collection of autonomous computers interconnected by a single technology; Internet as interconnected networks; the Internet as the most well-known internetwork; distributed system as independent computers appearing as a coherent system.
- **Uses of Computer Networks:** Business applications (resource sharing, remote access, client-server model, data sharing, VoIP, IP telephone, desktop sharing, e-commerce); home applications (access to remote info, instant messaging, social networks, electronic commerce, entertainment, ubiquitous computing); mobile users (wireless networks, GPS, m-commerce, sensor networks, wearable computers); social issues.
- **Network Hardware:** Classification by scale (PAN, LAN, MAN, WAN, internetworks); transmission technologies (broadcast links, point-to-point links); packet transferring modes (broadcast, multicast, unicast, anycast, geocast).
- **Network Software:** Protocol hierarchies (peers, protocols, interfaces, protocol stack, network architecture); design issues (reliability - error control, routing; protocol layering - addressing/naming, internetworking, scalable; resource allocation - statistical multiplexing, flow control, congestion control, QoS; security - confidentiality, authentication, integrity, nonrepudiation); connection-oriented vs connectionless service; packet switching modes (store-and-forward, cut-through); service primitives (LISTEN, CONNECT, ACCEPT, RECEIVE, SEND, DISCONNECT); relationship of services to protocols.
- **Reference Models:** OSI RM (7 layers: Physical, Data Link, Network, Transport, Session, Presentation, Application; PDUs: bit stream, frame, packet, segment/datagram, SPDU, PPDU, message); TCP/IP RM (4 layers: Link, Internet, Transport, Application); hybrid model used in book (5 layers: Physical, Data Link, Network, Transport, Application); comparison of OSI and TCP/IP (similarities: stacked protocols, similar functionality in layers, end-to-end transport; differences: OSI emphasizes services/interfaces/protocols, ordering, connection modes); critiques of OSI (bad timing, technology, implementations, politics) and TCP/IP (blurs services/interfaces/protocols, not general, link layer not true layer, doesn't distinguish physical/data link).
- **Example Networks:** The Internet (origin from ARPANET, IMPs, NSFNET, architecture with ISPs, IXPs, backbones); 3G mobile phone networks (AMPS, GSM, UMTS/WCDMA, LTE); wireless LANs (802.11); RFID and sensor networks.
- **Network Standardization:** Important organizations (ISO, ITU, IEEE, IETF, W3C).
- **Metric Units:** Use of SI prefixes (kilo, mega, giga, etc.) for network speeds.

### Detailed Explanations for Important Points

#### 1. Computer Network vs Distributed System
- **English:** A computer network is a collection of autonomous computers interconnected by a single technology. A distributed system is a collection of independent computers that appears to its users as a single coherent system.
- **中文:** 计算机网络是由单一技术互连的自治计算机的集合。分布式系统是独立计算机的集合，对其用户来说看起来像一个单一的连贯系统。

#### 2. OSI Reference Model
- **English:** The OSI model is a 7-layer framework for network architecture: Layer 1 (Physical) handles bit transmission; Layer 2 (Data Link) provides error-free transfer over a link; Layer 3 (Network) routes packets across networks; Layer 4 (Transport) provides end-to-end reliable delivery; Layer 5 (Session) manages dialog control; Layer 6 (Presentation) handles data syntax; Layer 7 (Application) supports user applications. It separates services, interfaces, and protocols, with PDUs defined for each layer.
- **中文:** OSI 模型是一个7层网络架构框架：层1 (物理层) 处理比特传输；层2 (数据链路层) 提供链路上的无错传输；层3 (网络层) 跨网络路由数据包；层4 (传输层) 提供端到端可靠交付；层5 (会话层) 管理对话控制；层6 (表示层) 处理数据语法；层7 (应用层) 支持用户应用。它分离服务、接口和协议，每层定义PDU。

#### 3. TCP/IP Reference Model
- **English:** The TCP/IP model has 4 layers: Link (hardware interface), Internet (IP for routing), Transport (TCP for reliable, UDP for unreliable), Application (HTTP, SMTP, etc.). It is practical for the Internet but less general.
- **中文:** TCP/IP 模型有4层：链路层 (硬件接口), 互联网层 (IP 用于路由), 传输层 (TCP 用于可靠, UDP 用于不可靠), 应用层 (HTTP, SMTP 等)。它适用于互联网但不那么通用。

#### 4. Connection-Oriented vs Connectionless Service
- **English:** Connection-oriented service establishes a connection, uses it for data transfer, and releases it, similar to a telephone call. Connectionless service sends independent messages (datagrams) with full addressing, similar to mailing letters.
- **中文:** 面向连接的服务建立连接、使用它进行数据传输并释放它，类似于电话呼叫。无连接的服务发送带有完整地址的独立消息 (数据报), 类似于邮寄信件。

## Chapter 2: The Physical Layer

### Key Knowledge Points
- **Theoretical Basis:** Baseband vs bandpass signals; Nyquist theorem (noiseless channel rate); Shannon theorem (noisy channel capacity).
- **Transmission Media:** Guided (twisted pair - UTP categories, 568A/B standards; coaxial; fiber - multimode vs single-mode, refractive index, LED vs laser, fiber vs copper); unguided (electromagnetic spectrum, radio, microwave, infrared, satellite vs fiber).
- **Digital Modulation:** Baseband encodings (NRZ, Manchester, 4B/5B, etc.); baud rate vs bit rate; bandpass modulation (ASK, FSK, PSK, QAM); Gray code.
- **Multiplexing:** FDM, TDM, CDM (features, CDM detection/calculation).
- **Switching:** Circuit switching (setup, use, teardown); packet switching (store-forward); comparison (timing, features).

### Detailed Explanations for Important Points

#### 1. Nyquist Theorem
- **English:** For a noiseless channel with bandwidth H Hz and V discrete levels, the maximum data rate is 2H log2(V) bits/sec. This limits the signaling rate based on bandwidth.
- **中文:** 对于带宽为 H Hz、V 个离散电平的无噪声信道，最大数据率为 2H log2(V) 比特/秒。这基于带宽限制信令速率。

#### 2. Shannon Theorem
- **English:** For a channel with bandwidth H Hz and signal-to-noise ratio S/N, the maximum capacity is H log2(1 + S/N) bits/sec. This accounts for noise in real channels.
- **中文:** 对于带宽为 H Hz、信噪比为 S/N 的信道，最大容量为 H log2(1 + S/N) 比特/秒。这考虑了真实信道的噪声。

#### 3. Multimode vs Single-Mode Fiber
- **English:** Multimode fiber has a larger core, allows multiple light paths, cheaper but higher dispersion, for short distances. Single-mode has small core, single path, low dispersion, for long distances.
- **中文:** 多模光纤核心较大，允许多条光路，较便宜但色散较高，用于短距离。单模核心小，单条光路，色散低，用于长距离。

#### 4. CDMA (Code Division Multiple Access)
- **English:** Users share bandwidth using orthogonal codes; signal detection by correlating with code. Calculation: Chip sequences, inner product for orthogonality.
- **中文:** 用户使用正交码共享带宽；信号检测通过与码相关。计算：芯片序列，内积用于正交性。

## Chapter 3: The Data Link Layer

### Key Knowledge Points
- **Main Tasks and Services:** Error detection/correction, flow control, framing; services to network layer (unacknowledged connectionless, acknowledged connectionless, connection-oriented).
- **Framing:** Character count, byte stuffing, bit stuffing, coding violations.
- **Error Control:** Error-detecting/correcting codes, codewords, coding rate, Hamming distance (for detection/correction), parity bits determination, Hamming code encoding, interleaving code, checksums, CRC principle and calculation.
- **Flow Control:** Stop-wait protocols (utopia, no noise, with noise); sliding window (1-bit, go-back-n, selective repeat); delay-bandwidth product, pipelining.
- **Protocols:** PPP (frame format, state diagram).

### Detailed Explanations for Important Points

#### 1. Framing Methods
- **English:** Character count: A field in the header specifies the number of characters in the frame. Byte stuffing: Special flag bytes are stuffed with escape characters if they appear in data. Bit stuffing: Insert a 0 after five 1s in bit-oriented framing. Coding violations: Use invalid signal levels to mark frame ends in physical coding.
- **中文:** 成帧方法：字符计数：头部字段指定帧中的字符数。字节填充：如果数据中出现特殊标志字节，则用转义字符填充。比特填充：在面向比特的成帧中五个1后插入0。编码违例：在物理编码中使用无效信号电平标记帧结束。

#### 2. CRC Calculation
- **English:** CRC uses polynomial division: Append k zeros to message (for degree k polynomial), divide by generator polynomial, subtract remainder to get checksum. Receiver divides received frame; zero remainder means no error.
- **中文:** CRC 使用多项式除法：将消息附加 k 个零 (对于度 k 多项式), 除以生成多项式, 减去余数得到校验和。接收方除接收帧；余数零表示无错误。

#### 3. Sliding Window Protocol
- **English:** Allows multiple frames outstanding; sender has window of frames to send, receiver acks, window slides. Go-back-n: Retransmit from error. Selective repeat: Retransmit only error frame.
- **中文:** 滑动窗口协议：允许多个帧未确认；发送方有窗口发送帧, 接收方确认, 窗口滑动。回退N帧：从错误重传。选择性重传：仅重传错误帧。

## Chapter 4: The Medium Access Control Sublayer

### Key Knowledge Points
- **MAC Concepts:** MAC position in stack; collision; continuous/slotted transmission; carrier sensing/non-sensing; dynamic channel allocation assumptions.
- **Multiple Access Protocols:** ALOHA (pure, slotted); CSMA (1-persistent, non-persistent, p-persistent); comparisons; collision detection; collision-free (bit-map, token ring, binary countdown).
- **Wireless Issues:** Hidden/exposed stations; MACA.
- **CSMA/CD:** Work flow, binary exponential backoff.
- **Ethernet:** Frame format (IEEE 802.3), collision detection time, workflow.
- **Wireless LANs:** 802.11 modes (ad hoc, infrastructure); DCF/PCF; NAV virtual carrier sensing; frame intervals; frame format.
- **Bridges/Switches:** Differences from hubs; learning bridge; cut-through switching; spanning tree bridges; network devices by layer.

### Detailed Explanations for Important Points

#### 1. CSMA Variants
- **English:** 1-persistent CSMA: Listen, transmit if idle, wait if busy. Non-persistent: Backoff if busy. p-persistent: Transmit with probability p if idle. Comparisons: 1-persistent has higher throughput but more collisions; non-persistent reduces collisions but delay; p-persistent balances.
- **中文:** CSMA 变体：1-坚持 CSMA：监听, 空闲则传输, 忙则等待。非坚持：忙则退避。p-坚持：空闲则以概率 p 传输。比较：1-坚持吞吐量高但碰撞多；非坚持减少碰撞但延迟；p-坚持平衡。

#### 2. CSMA/CD
- **English:** Carrier Sense Multiple Access with Collision Detection: Stations listen before and during transmission; if collision, abort and backoff using binary exponential algorithm (slot time, retry limit).
- **中文:** 载波侦听多路访问带碰撞检测：站传输前和期间监听；如果碰撞, 中止并使用二进制指数退避算法 (时隙时间, 重试限)。

#### 3. Ethernet Frame Format
- **English:** Preamble (7 bytes sync, 1 byte start), destination/source address (6 bytes each), type/length (2 bytes), data (46-1500 bytes), CRC (4 bytes).
- **中文:** 以太网帧格式：前导码 (7 字节同步, 1 字节开始), 目的/源地址 (各 6 字节), 类型/长度 (2 字节), 数据 (46-1500 字节), CRC (4 字节)。

#### 4. Spanning Tree Bridges
- **English:** Bridges form a tree to avoid loops; root bridge elected, ports designated/blocked based on cost.
- **中文:** 生成树网桥：网桥形成树避免环路；选举根网桥, 基于成本指定/阻塞端口。
